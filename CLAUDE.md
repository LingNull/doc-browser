# CLAUDE.md — 文档浏览系统

## 项目概述

这是一个**轻量级文档浏览与详设生成系统**，用于管理开发过程中产生的各类文档。

- **启动**: `node server.js` → 浏览器打开 `http://localhost:3000`
- **技术栈**: Node.js 零依赖服务器 + 纯 HTML/JS/jQuery 前端
- **目录**: `server.js` (服务端), `public/` (前端), `docs/` (文档根目录)

## 文档归档约定

项目使用 `docs/{项目名}/{模块名}/{功能名}/` 的三级目录结构管理所有文档。

### 目录结构
```
docs/
└── {项目名}/              # 如：电商平台、后台管理系统
    └── {模块名}/          # 如：用户模块、订单模块
        └── {功能名}/      # 如：登录功能、注册功能
```

### 文档类型与文件名
每次完成开发任务后，AI 助手应将生成的文档直接写入对应的功能目录：

| 文档类型 | 文件名 | 说明 |
|---------|--------|------|
| 需求理解/分析 | `需求文档.md` | 功能需求、非功能需求、验收标准 |
| 开发计划/方案 | `开发计划.md` | 技术方案、排期、风险评估 |
| 代码审查报告 | `代码审查.md` | 问题发现、改进建议、审查结论 |
| 测试方案/报告 | `测试报告.md` | 测试用例、兼容性、遗留问题 |
| API 接口文档 | `接口文档.md` | 接口定义、参数说明、示例 |
| 数据库设计 | `数据库设计.md` | ER 图、表结构、索引设计 |
| 详细设计文档 | `详细设计文档.md` | 系统生成的综合设计文档 |

### 行为规则（AI 助手必须遵守）

1. **生成任何文档时，必须先确认**：属于哪个项目？哪个模块？哪个功能？
   - 先扫描 `docs/` 下已有的项目，列出让用户选择
   - 如果用户未明确指定，**主动询问**目标项目/模块/功能
2. **对应目录不存在时，自动创建**完整的三级目录链
3. **写入后告知用户**在浏览器中查看：`http://localhost:3000`
4. **文件名约定**：使用上述表格中的标准文件名

### 操作示例

```
用户：帮我写一份登录功能的需求文档
AI：  docs/ 下已有项目：[示例项目]，请问是哪个项目？
用户：示例项目
AI：  示例项目下已有模块：[用户模块] [订单模块]，请问是哪个模块？
用户：用户模块
AI：  好的，已生成需求文档，保存至 docs/示例项目/用户模块/登录功能/需求文档.md
     可在 http://localhost:3000 中查看
```

## 服务端 API

| 方法 | 路由 | 说明 |
|------|------|------|
| GET | `/api/tree` | 获取目录树 JSON |
| GET | `/api/tree?path=...` | 获取子树 |
| GET | `/api/file?path=...` | 读取文件内容 |
| POST | `/api/generate` | 生成详细设计文档 |
| POST | `/api/save` | 保存文件到 docs/ |

## 前端页面

- 左右分栏布局：左侧目录树 + 右侧内容区
- 支持 .md 渲染和 .txt 展示
- 右键功能目录 → "生成详细设计"可聚合该功能下所有文档
- 详设生成器支持编辑/预览/保存
