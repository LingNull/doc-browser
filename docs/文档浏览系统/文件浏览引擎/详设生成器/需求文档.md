# 详设生成器 — 需求文档

## 功能概述

基于某个功能目录下的所有文档，自动聚合生成一份结构化的"详细设计文档"。

## 功能需求

### FR-01: 文档聚合
- 读取目标目录下所有 `.md` / `.txt` 文件
- 按文件名关键词智能分类到对应章节
- 调用 `POST /api/generate` 获取生成结果

### FR-02: 章节模板
- 10 个预定义章节：概述、功能需求、流程图、实现逻辑、核心逻辑、数据表结构、接口设计、风险评估、测试方案、部署说明
- 空章节显示"待补充"提示

### FR-03: 编辑/预览切换
- 编辑模式：每个章节一个 `<textarea>`，可自由编辑
- 预览模式：实时渲染编辑后的 Markdown
- 切换按钮高亮当前模式

### FR-04: 保存
- 调用 `POST /api/save` 写入文件
- 保存到当前功能目录下的 `详细设计文档.md`
- 成功后自动刷新目录树

## 文档分类规则

| 关键词 | 对应章节 |
|--------|----------|
| 需求/requirement | 功能需求 |
| 计划/plan | 功能实现逻辑 |
| 审查/review | 核心逻辑 |
| 测试/test | 测试方案 |
| 数据/database/db/表 | 数据表结构 |
| API/接口 | 接口设计 |
| 风险/risk | 风险评估 |
| 部署/deploy | 部署说明 |
