# 详设生成器 — 开发计划

## 技术方案

### 服务端（server.js）
- `POST /api/generate`：文档收集 → 分类 → 合并 → 返回结构化 Markdown
- `POST /api/save`：保存生成文档到磁盘
- `classifyDocument()`：文件名关键词匹配
- `generateDesignDoc()`：核心生成逻辑
- `assembleMarkdown()`：组装最终 Markdown

### 前端（generator.js）
- 编辑模式：章节卡片 + textarea
- 预览模式：marked.js 渲染
- 模式切换和保存交互
