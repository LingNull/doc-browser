# Save Doc Skill

交互式文档归档 Skill。用户调用 `/save-doc` 后，AI 助手按以下流程操作：

## 流程

### Step 1: 扫描已有项目
用 Glob 扫描 `docs/` 下的一级子目录（即项目），列出供用户选择。

### Step 2: 确认或询问
- 从用户输入中提取：项目名、模块名、功能名、文档类型
- 对于缺失的信息，列出已有选项 + 允许新建

### Step 3: 创建目录
对应 `docs/{项目}/{模块}/{功能}/` 目录不存在时，自动创建完整目录链。

### Step 4: 生成文档
根据文档类型选择标准文件名：
| 关键词 | 文件名 |
|--------|--------|
| 需求/requirement | 需求文档.md |
| 计划/plan | 开发计划.md |
| 审查/review | 代码审查.md |
| 测试/test | 测试报告.md |
| 接口/API | 接口文档.md |
| 数据库/DB | 数据库设计.md |

### Step 5: 写入并告知
- 将生成的内容写入 `docs/{项目}/{模块}/{功能}/{文件名}`
- 告知用户保存路径和浏览器查看地址：`http://localhost:3000`
